# Ember Vanguard — 作品介绍

> **ARKHAM COUNTY ARCHIVE · CASE 23-07 / OCTOBER 17, 1923**  
> 克苏鲁风格 AI 驱动文字冒险游戏

---

## 项目概述

**Ember Vanguard**（灰石庄园调查档案）是一款运行于 Web 浏览器的克苏鲁风格文字冒险游戏。玩家扮演 1923 年的调查者，独闯新英格兰郊外的废弃庄园，在 AI 实时生成的叙事中探索、抉择、挣扎求生。

游戏的核心命题是：**"未知比已知更恐怖。"** 所有骰子在后台暗骰，AI 只呈现文学化结果；地图永不重复，每局故事从不相同；理智与体力同等重要，结局由你的每一个选择积累而来。

---

## AI 创新

### 1. AI 作为实时叙事引擎

游戏接入本地部署的大语言模型（Ollama / llama3.1:8b），每个房间的叙事文本、遭遇事件、行动选项均由 AI **实时生成**，而非从预设文本库中抽取。

- AI 扮演"地下城主（DM）"角色，根据玩家当前坐标、背景职业、状态数值、持有物品动态裁定房间内容
- 系统 Prompt 精心设计，强制 AI 输出结构化 JSON（含叙事文字、HP/SAN 变动、新物品、行动选项），主程序解析后驱动状态机，实现**叙事与数值的无缝结合**
- System Prompt 严格控制在 500 tokens 以内，每次只传当前房间上下文 + 玩家状态摘要，避免 context 膨胀

```json
{
  "narration": "画像中的人突然发出一声尖叫！你的精神受到了剧烈冲击。",
  "hp_delta": 0,
  "san_delta": -2,
  "new_item": null,
  "is_cursed": false,
  "options": [
    {"key": "1", "text": "仔细检查画像"},
    {"key": "2", "text": "转身离开这个房间"},
    {"key": "3", "text": "用手帕遮住眼睛继续前进"}
  ]
}
```

### 2. AI 驱动个性化结局

游戏结束时，AI 接收玩家的**完整状态快照**（角色背景、最终 HP/SAN、物品清单、关键事件列表、探索房间数、结局类型），生成 200–400 字的专属结局叙事。两次游戏即便走向相同的结局类型，文字也绝对不同。

### 3. 异变期：AI 接管终局

当诅咒值积累到阈值，AI 完全接管游戏进入"异变期"——地图探索挂起，AI 临时扮演"远古存在"，根据玩家当前拥有的物品和已探索的房间，即时生成专属的终极危机（Boss 战或逃生任务），进入生死存亡的连续判定阶段。

### 4. 容错 Fallback 机制

AI 不可用时（网络异常、模型超时），系统自动根据坐标哈希选取 5 种预设 Fallback 事件，保证游戏**永不崩溃**，随时可玩。

## 核心游戏机制

### 角色背景系统

开局从三个背景中选择，影响初始属性与 AI 叙事风格：

| 背景 | HP | SAN | 特殊能力 |
|------|----|-----|---------|
| **记者** THE JOURNALIST | 8 | 7 | 每进入新房间额外获得一条碎片线索，辅助 AI 叙事分支 |
| **医生** THE PHYSICIAN | 7 | 9 | 首次 SAN 归零时触发"理性抵抗"豁免，SAN 保留为 1 |
| **盗墓者** THE TOMB RAIDER | 10 | 5 | 战斗检定 +1 加成，但初始理智最低，更易触发异变 |

### 坐标哈希地图

地图采用坐标哈希表实现无限边界按需生成：

- 玩家从 `(0, 0)` 出发，向四个方向探索，地图无硬性边界
- 同一坐标永远生成同一房间（确定性哈希），支持回探，保持世界内部一致性
- 无需预先存储整张地图，规避数组越界，极大降低工程复杂度

```python
# 前端坐标哈希示例
function coordinateHash(x, y, salt) {
  let h = (x * 374761393 + y * 1120534327 + salt) | 0;
  h = Math.imul(h ^ (h >>> 16), 0x45d9f3b);
  return (h ^ (h >>> 16)) >>> 0;
}
```

### 隐藏 D20 检定

所有行动均触发后台 D20 骰点，玩家**永远看不到骰子数字**：

```python
def skill_check(stat_value: int, difficulty: int = 10) -> tuple[bool, int]:
    roll = random.randint(1, 20)
    modified = roll + (stat_value // 3) + background_bonus
    return modified >= difficulty, roll
```

- 三个选项对应三级难度（稳妥 10 / 冒险 11 / 鲁莽 12），越冒险的选项收益越高
- 系统将骰点结果、玩家属性、行动描述一并传给 AI，AI 据此生成成功或失败的文学化叙事
- 检定类型智能推断：关键词匹配"攻击/砸/踢" → 战斗检定；"观察/阅读/聆听" → 精神检定

### 双阶段游戏循环

**探索期**
- 在坐标地图上自由移动（WASD / 方向键）
- 进入每个新房间触发随机遭遇（AI 生成），必须处理后才能继续移动
- 遭遇类型按概率分布：普通事件（43%）、物品事件（22%）、受诅物事件（18%）、逃脱出口（≥6 间后约 3.5%）

**异变期**
- 触发条件 A：诅咒检定，概率随诅咒等级指数上升（5% → 20% → 45% → 80%）
- 触发条件 B：强制触发——已探索房间达 20 间
- AI 接管战局，进行连续三轮判定：①利用线索寻找封印缺口 ②布置临时防线 ③强行打断仪式

```python
def curse_check(curse_level: int) -> bool:
    threshold = min(curse_level ** 2 * 5, 80)  # 上限 80%
    return random.randint(1, 100) <= threshold
```

### 五重结局系统

| 结局 | 触发条件 | 基调 |
|------|---------|------|
| **逃脱·理智尚存** | HP > 0，SAN > 3，持有逃脱出口 | 惊魂后的生还，带走了部分真相 |
| **逃脱·精神破碎** | HP > 0，SAN ≤ 3，持有逃脱出口 | 肉体逃离，理智已被侵蚀 |
| **融合·成为祭品** | SAN = 0，HP > 0 | 主动为远古存在打开封印，失去自我 |
| **战死·无名英雄** | HP = 0，诅咒 < 3 | 死于探索，庄园秘密永远封存 |
| **湮灭·彻底消失** | HP = 0，诅咒 ≥ 3，或异变期战败 | 最坏结局，连尸骨都找不到 |

程序只判断走向哪条结局路径，具体文字由 AI 根据玩家全程状态快照生成，保证每局唯一。

## 技术架构

### 双路架构

| 层级 | 技术 | 说明 |
|------|------|------|
| **Web 前端** | Next.js 15 + React 19 + TypeScript | 纯客户端游戏引擎，离线完整可玩 |
| **Python 后端** | FastAPI + Uvicorn | REST API 服务，提供 AI 增强叙事 |
| **AI 模型** | Ollama (llama3.1:8b) | 本地部署，叙事质量与速度平衡 |
| **边缘部署** | Cloudflare Workers | 全球低延迟分发 |
| **状态持久化** | localStorage | 自动存档，随时关闭随时继续 |

### 前端架构亮点

- **纯函数状态管理**：所有游戏函数均为纯函数（`structuredClone` 深拷贝），状态完全可追踪、可回放，无副作用
- **打字机叙事节奏**：每字符精心设计停顿（标点 150ms、逗号 72ms、普通字符 15ms），将恐怖感融入阅读节奏；打字期间屏蔽所有操作交互
- **自定义 `useGame` Hook**：统一管理键盘事件监听、localStorage 存档读写、游戏阶段流转

### AI Pipeline

```
玩家操作 → FastAPI → skill_check(D20) → AIPipeline.generate_room()
                                                  ↓
                                        Ollama llama3.1:8b
                                                  ↓
                                       JSON 叙事 + 选项
                                         （Fallback 容错）
```

## 世界观摘录

> 铁门在你身后轰然关闭。  
> 黑暗吞没了来时的路。  
> 空气中有一股腐烂的甜味——  
> 像是某种东西正在等待你。  
>
> 你握紧手中的手电筒，向前走去。

部分房间意象：

- **肖像画廊**：所有画像眼睛被涂黑，最末一幅是玩家自己的脸
- **地下酒窖**：一瓶标签写着明天日期的酒，瓶底字母拼出的姓氏与玩家相同
- **儿童房**：发条小车绕着空摇篮转，墙上刻着玩家的名字
- **积水浴室**：水中倒影比玩家多出一只手
- **废弃观测台**：星图标注的是庄园地下房间的分布，而非天空
